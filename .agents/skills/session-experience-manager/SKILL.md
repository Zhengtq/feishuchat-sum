# Session Experience Manager (会话经验管理器)

## ⚙️ 核心配置 (Configuration)
> **⚠️ 重要指令**：大模型在此技能执行期间的所有读取、写入经验文档的操作，**必须**依赖以下基础路径变量：
> `EXPERIENCE_BASE_PATH = "/Users/zhengtq/我的云端硬盘/coding_project/experience"`

## 核心目标
解决大模型每次开启新 Session 就丢失项目历史上下文与“踩坑”记忆的问题。通过：
1. 结构化的路由文档(`router.md`)和具体的经验文档(`single_experience/*.md`)实现经验的低成本持久化。
2. 维护项目根目录下的全局信息文件（`PROJECT_ALL_INFO.md`）保持项目历史脉络的延续。

## 何时触发？(Triggers)
1. **获取经验与上下文 (Retrieve)**：当用户开启全功能新的会话，或者遇到复杂依赖、难调 Bug、或对某个领域没有头绪时。 -> [查看如何检索经验](references/retrieve_experience.md)
2. **沉淀经验与更新全局信息 (Save)**：当任务完成，且过程包含显著的踩坑、多步排查，或者对项目引入了**新的需求、重要开发变更、技术选型**时。 -> [查看如何保存经验](references/save_experience.md)

## 工作模式：渐进式披露与按需读取
不要一次性加载所有历史记录。始终遵循以下原则：
1. **全局概貌**：优先检查并阅读当前工作目录根目录下的 `PROJECT_ALL_INFO.md`，建立对项目历史变迁的基本认知。
2. **先看路由**：如果是解决具体技术问题，先查阅全局索引 `${EXPERIENCE_BASE_PATH}/router.md`。
3. **再看细节**：只有当 `router.md` 中提及的某个条目与当前问题高度相关时，才去读取对应的 `${EXPERIENCE_BASE_PATH}/single_experience/具体文档.md`。
4. **保持分离**：保存记录时，严格区分什么是“踩坑经验分享”，什么是“项目需求变更纪要”。
